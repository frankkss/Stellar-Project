PROJECT NAME: Decentralized Secure File Sharing  

PROBLEM:  
Businesses risk losing access, privacy, and control when storing files on centralized platforms like Google Drive or Dropbox.  

SOLUTION:  
Our dApp uses Stellar Soroban contracts to store files across a decentralized network, enforce permission controls, and log tamper‑proof access records.  

STELLAR FEATURES USED:  
[ ] XLM/USDC transfer  
[ ] Custom tokens  
[x] Soroban contract  
[ ] Built-in DEX  
[ ] Trustline  
[ ] Clawback/Compliance  

TARGET USERS:  
Businesses that need secure, transparent, and reliable file sharing.  

CORE FEATURE (MVP):  
One transaction that uploads a file, scans it for safety, and records its unique ID with permission rules on Stellar.  

WHY STELLAR:  
Traditional cloud storage depends on one company and lacks transparent records; other chains are costly and complex. Stellar’s Soroban contracts provide low‑cost, fast, and verifiable transactions with built‑in trust.
